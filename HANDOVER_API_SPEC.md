## 1. Trace Injection

trace_id must be provided by the caller on every request to Core (web1).
It flows through all layers unchanged.

**Header:**
```
X-TRACE-ID: <uuid>
```

**Example:**
```
X-TRACE-ID: 5d050c8c-c880-4e6d-9a01-8274556f30ec
```

**Rules:**
- If missing → request rejected with `400 trace_id required`
- If `"auto"` → web1 generates a UUID automatically (for browser use only)
- Never generated by Monitor, Sarathi, or Executer

## 2. Core — Login Endpoint

```
POST http://54.156.236.10:30001/login
```

**Required header:**
```
X-TRACE-ID: <uuid>
```

**Required form field:**
```
user_id=<string>
```

**Full curl:**
```bash
curl -X POST http://54.156.236.10:30001/login \
  -H "X-TRACE-ID: 5d050c8c-c880-4e6d-9a01-8274556f30ec" \
  -d "user_id=test"
```

**Response (HTML):**
```html
<h2>Dashboard - Web1</h2>
<p>Welcome test</p>
<p><b>Trace ID:</b> 5d050c8c-c880-4e6d-9a01-8274556f30ec</p>
...
```

**Side effects:** Posts 3 events to Monitor: `session_start`, `user_login`, `page_view`

## 4. Monitor — Event Ingestion

```
POST http://54.156.236.10:30004/track-event
```

**This endpoint is called by web1, sarathi, and executer — not by humans directly.**

**Required JSON fields:**
```json
{
  "user_id":    "<string>",
  "event_type": "<string>",
  "timestamp":  <unix int>,
  "session_id": "<string>",
  "trace_id":   "<uuid string>"
}
```

**Optional fields (signal-specific):**
```json
{
  "metadata":     {"source": "web1"},
  "execution_id": "<uuid>",
  "service":      "<string>",
  "action":       "<string>",
  "status":       "success | failed",
  "result":       "SUCCESS | FAILURE",
  "enforcement_status": "validated"
}
```

**Response (success):**
```json
{"status": "ok"}
```

**Response (missing required fields):**
```json
{"error": "invalid event"}
```
HTTP 400

## 5. Sarathi — Decision Endpoint

```
POST http://54.156.236.10:30005/decision
```

**Required JSON body:**
```json
{
  "trace_id":    "<uuid>",
  "service_id":  "<string>",
  "action_type": "<string>",
  "payload": {
    "decision_score": <float 0.0–1.0>
  }
}
```

**Full curl:**
```bash
curl -X POST http://54.156.236.10:30005/decision \
  -H "Content-Type: application/json" \
  -d '{
    "trace_id": "5d050c8c-c880-4e6d-9a01-8274556f30ec",
    "service_id": "web1-blue",
    "action_type": "restart",
    "payload": {"decision_score": 0.9}
  }'
```

**Policy logic:**
- `score > 0.6` → ALLOW → calls Executer
- `score > 0.35` → ESCALATE → no execution
- `score <= 0.35` → BLOCK → no execution

**Response (ALLOW — real output):**
```json
{
  "status": "executed",
  "trace_id": "5d050c8c-c880-4e6d-9a01-8274556f30ec",
  "executer_response": {
    "execution_id": "4a8e2bb4-bb4a-427e-a142-0f0e8e69eb9d",
    "latency": 0.6184470653533936,
    "output": "deployment.apps/web1-blue patched",
    "status": "success",
    "trace_id": "5d050c8c-c880-4e6d-9a01-8274556f30ec"
  }
}
```

**Response (BLOCK):**
```json
{"status": "BLOCK", "trace_id": "<trace_id>"}
```

**Response (missing trace_id):**
```json
{"error": "trace_id required"}
```
HTTP 400

## 6. Executer — Execute Action Endpoint

```
POST http://54.156.236.10:30003/execute-action
```

**This endpoint is only callable by Sarathi. Direct calls return 403.**

**Required header:**
```
X-CALLER: sarathi
```

**Required JSON body:**
```json
{
  "trace_id":   "<uuid>",
  "service_id": "<string>",
  "action":     "<string>"
}
```

**Allowed service_id values:**
```
web1-blue, web1-green, web2-blue, web2-green
```

**Allowed action values:**
```
restart, scale
```

**Response (success — real output):**
```json
{
  "execution_id": "4a8e2bb4-bb4a-427e-a142-0f0e8e69eb9d",
  "latency": 0.6184470653533936,
  "output": "deployment.apps/web1-blue patched",
  "status": "success",
  "trace_id": "5d050c8c-c880-4e6d-9a01-8274556f30ec"
}
```

**Response (missing X-CALLER header):**
```json
{"error": "unauthorized"}
```
HTTP 403

**Response (invalid service):**
```json
{
  "execution_id": "8ca09b1b-a5b0-457e-8ef0-5d6bb6cc2e85",
  "status": "failed",
  "error": "invalid service"
}
```

## 7. Monitor — SSE Stream

```
GET http://54.156.236.10/signals/stream
GET http://pravah.blackholeinfiverse.com/signals/stream
```

**No headers required. No body.**

**Full curl:**
```bash
curl -H "Host: pravah.blackholeinfiverse.com" \
  -N http://54.156.236.10/signals/stream
```

**Output format — one signal per SSE event:**
```
data: {<flat signal JSON>}

: keepalive
```

**Signal fields — always present:**
```
signal_type   string   — type of event
service       string   — which service emitted it
metric        string   — always "status"
value         string   — "RUNNING", "SUCCESS", or "FAILURE"
severity      string   — "INFO" or "CRITICAL"
timestamp     int      — unix epoch
trace_id      string   — original trace_id from Core
trace_origin  string   — always "core"
trace_hash    string   — SHA-256 of trace_id
source        string   — always "core"
emitted_at    string   — ISO8601 UTC timestamp
```

**Signal fields — present only when applicable:**
```
execution_id        string   — on execution, verification, execution_completed
decision            string   — on decision signal (ALLOW/BLOCK/ESCALATE)
policy_reference    string   — on decision signal
action              string   — on decision signal
enforcement_status  string   — on enforcement signal ("validated")
result              string   — on verification signal (SUCCESS/FAILURE)
```

**Stream stays open indefinitely. Keepalives sent every ~0.2s when queue is empty.**
